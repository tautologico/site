Linguagens e Modelos para Programação na GPU

Introdução

As placas gráficas dos computadores pessoais atuais surgiram com o propósito de renderizar gráficos em tempo-real, principalmente para jogos. Uma placa gráfica inclui um ou mais processadores gráficos (GPU ou Graphics Processing Unit), que pode realizar algumas operações muito mais rápido que o processador central de um computador (CPU).

Atualmente, as GPUs são programáveis e existe um grande interesse em usá-las em aplicações gerais, não apenas restritas à geração de gráficos em tempo-real. Uma boa placa gráfica é otimizada para realizar um grande número de operações com números de ponto-flutuante por segundo, o que as torna ideais para aplicações de alta demanda computacional em domínios como processamento multimídia (imagens, vídeo) e ciência computacional (simulação, resolução de modelos matemáticos).

Um problema para o uso das placas gráficas para aplicações gerais é que o modelo computacional das GPUs é bastante diferente do modelo de máquina de Von Neumann com o qual os programadores estão acostumados na CPU. Existem sistemas criados para o propósito de usar as GPUs para programação geral, como CUDA da nVidia e o OpenCL. Entretanto, esses sistemas se mantêm ligados a linguagens de programação criadas para a programação de CPUs: tanto a linguagem CUDA C da nVidia quanto a linguagem associada ao OpenCL são variantes da linguagem C para acessar GPUs.

Objetivos

Os objetivos deste projeto são investigar modelos computacionais de computação paralela em GPUs, e criar ou sugerir linguagens de programação, associados a esses modelos, que sejam mais adequadas para a programação de GPUs. Na definição dos modelos, uma aplicação interessante é identificar quando uma aplicação tem o potencial de obter grandes ganhos de eficiência se executada na GPU ao invés da CPU.

Os modelos e linguagens desenvolvidos devem ser usados na criação de aplicações de exemplo que demonstrem a maior facilidade de programação.
